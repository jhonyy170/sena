1. Conceptos de Bases de Datos

Una base de datos es un sistema que almacena datos organizados y relacionados entre sí para facilitar su acceso y gestión.

Desde el punto de vista informático, está compuesta por tablas que contienen columnas (atributos) y filas (registros).

2. Características de una Base de Datos

  • Independencia lógica y física de los datos.
  • Redundancia mínima.
  • Acceso concurrente para múltiples usuarios.
  • Integridad de los datos.
  • Consultas complejas optimizadas.
  • Seguridad de acceso y auditoría.
  • Respaldo y recuperación.
  • Compatibilidad con lenguajes estándar.

3. Ventajas de las Bases de Datos

Control sobre la redundancia de datos

Evita el almacenamiento innecesario de datos repetidos, optimizando el uso del espacio y mejorando la consistencia.

Consistencia de datos

Reduciendo la redundancia, se minimiza el riesgo de inconsistencias en la información almacenada.

Compartición de datos

Permite que múltiples usuarios accedan a la base de datos simultáneamente con los permisos adecuados.

Mantenimiento de estándares

Facilita el cumplimiento de estándares nacionales e internacionales en almacenamiento y gestión de datos.

Seguridad e integridad

Garantiza la validez y seguridad de los datos mediante restricciones y permisos de acceso.

4. Desventajas de las Bases de Datos

Complejidad

El uso de un SGBD requiere conocimientos técnicos avanzados para su correcta implementación y mantenimiento.

Coste del equipamiento adicional

El almacenamiento de grandes volúmenes de datos puede requerir hardware más potente y costoso.

Vulnerabilidad ante fallos

Al centralizar los datos en un sistema, un fallo puede afectar toda la base de datos, por lo que se recomienda el uso de backups.

5. Sistema de Gestión de Base de Datos (SGBD)

Un SGBD es un software especializado que gestiona el almacenamiento, manipulación y consulta de bases de datos.

Se dividen en:

  • Relacionales (SQL): Organizan la información en tablas y establecen relaciones entre ellas.
  • No relacionales (NoSQL): Almacenan datos en estructuras flexibles como documentos o grafos.

6. Bases de Datos Relacionales

Se basan en el modelo de organización en tablas con relaciones definidas. Sus características principales incluyen:

  • Atomicidad
  • Consistencia
  • Aislamiento
  • Durabilidad

Ejemplos: MySQL, Oracle, SQL Server, PostgreSQL.

7. Bases de Datos No Relacionales

No utilizan identificadores relacionales y almacenan datos en formatos más flexibles como documentos JSON.

Ejemplos: MongoDB, Redis, Elasticsearch, Cassandra.

1. Estructura de una Base de Datos

Campo o Atributo

Unidad básica de entrada de datos en un registro. Cada entrada puede tener múltiples campos de distintos tipos.

Registro

Una fila en una base de datos, que contiene datos únicos en cada campo.

Dato

Valor que pertenece a un campo dentro de un registro.

2. Tabla o Entidad

Unidad dentro de una base de datos que contiene atributos que la caracterizan.

Relación

Asociación entre entidades que permite definir dependencias y compartir atributos.

Tipos de Relaciones

Uno a Uno (1:1): Un registro de una entidad se asocia solo con un registro de otra.

Uno a Muchos (1:M): Un registro de una entidad se asocia con múltiples registros de otra.

Muchos a Muchos (N:M): Un registro de una entidad puede relacionarse con múltiples registros de otra.

3. Modelo Entidad Relación

Representación gráfica de las entidades y sus relaciones en una base de datos.

  • Entidades: Se representan con un rectángulo.
  • Atributos: Se representan con círculos o elipses y se conectan a su entidad.
  • Relaciones: Se representan con un rombo.

4. Modelo Relacional

Transformación del modelo entidad-relación a una estructura basada en tablas.

  1. Cada entidad se convierte en una tabla con sus atributos como campos.
  2. El atributo principal se convierte en llave primaria (PK).
  3. Las relaciones se establecen mediante llaves foráneas (FK).

5. Ejemplo de Base de Datos

Ejemplo de facturación en una tienda:

  • Un cliente puede tener múltiples facturas.
  • Una factura puede contener varios productos.
